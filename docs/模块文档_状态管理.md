# 🧠 状态管理模块文档

---

## 一、模块定位与职责

**模块名称**: State Management (状态管理模块)

**职责**:
- 全局状态对象管理
- 有限状态机(FSM)状态转移
- 会话锁机制（防止异步竞态）
- 输入锁机制（防止多键并弹）
- 定时器托管与生命周期管理
- localStorage持久化支持

**文件位置**: [js/state.js](js/state.js)

## 二、全局状态对象

### 2.1 状态结构

```javascript
export const state = {
    // 系统状态
    currentState: STATE.IDLE,
    currentMode: MODES.NORMAL,
    currentSpeed: 1,
    
    // 当前单词
    currentWord: null,
    
    // 历史记录
    historyStack: [],
    historyPointer: -1,
    
    // 统计数据
    todayCount: 0,
    knowCount: 0,
    unfamiliarCount: 0,
    combo: 0,
    
    // 会话锁（防止异步竞态）
    playSessionID: 0,
    
    // 输入锁（防止多键并弹）
    isProcessingInput: false,
    
    // 定时器托管桶
    activeTimersPool: [],
    
    // 不熟单词列表
    unfamiliarWords: []
};
```

### 2.2 状态字段详解

| 字段 | 类型 | 说明 | 初始值 |
|------|------|------|--------|
| `currentState` | String | 当前系统状态 | `IDLE` |
| `currentMode` | Number | 当前训练模式 | `1` (NORMAL) |
| `currentSpeed` | Number | 当前速度档位 | `1` |
| `currentWord` | Object | 当前显示的单词 | `null` |
| `historyStack` | Array | 历史单词栈 | `[]` |
| `historyPointer` | Number | 历史栈指针 | `-1` |
| `todayCount` | Number | 今日训练次数 | `0` |
| `knowCount` | Number | 标记"知道"的次数 | `0` |
| `unfamiliarCount` | Number | 标记"不熟"的次数 | `0` |
| `combo` | Number | 当前连击数 | `0` |
| `playSessionID` | Number | 会话锁ID | `0` |
| `isProcessingInput` | Boolean | 输入处理中标记 | `false` |
| `activeTimersPool` | Array | 活跃定时器ID列表 | `[]` |
| `unfamiliarWords` | Array | 不熟单词列表 | `[]` |

## 三、有限状态机(FSM)

### 3.1 状态定义

| 状态 | 值 | 描述 | 行为约束 |
|------|-----|------|----------|
| **IDLE** | `'IDLE'` | 空闲状态 | 可启动训练 |
| **PLAYING** | `'PLAYING'` | 播放中 | 显示单词，等待输入 |
| **WAIT_RESPONSE** | `'WAIT_RESPONSE'` | 等待响应 | 已显示释义，等待用户反馈 |
| **SHOW_ANSWER** | `'SHOW_ANSWER'` | 展示答案 | 已标记不熟，显示释义 |

### 3.2 状态转移矩阵

| 当前状态 | 触发事件 | 目标状态 | 说明 |
|----------|----------|----------|------|
| IDLE | TOGGLE_PLAY | PLAYING | 开始训练 |
| IDLE | SWITCH_MODE | IDLE | 切换模式（不改变状态） |
| PLAYING | TOGGLE_PLAY | IDLE | 暂停训练 |
| PLAYING | SHOW_MEANING | WAIT_RESPONSE | 显示释义 |
| PLAYING | UNFAMILIAR | SHOW_ANSWER | 标记不熟 |
| PLAYING | NEXT_WORD | PLAYING | 下一个单词 |
| WAIT_RESPONSE | TOGGLE_PLAY | IDLE | 暂停训练 |
| WAIT_RESPONSE | KNOW | PLAYING | 确认知道 |
| WAIT_RESPONSE | UNFAMILIAR | SHOW_ANSWER | 标记不熟 |
| WAIT_RESPONSE | NEXT_WORD | PLAYING | 下一个单词 |
| SHOW_ANSWER | TOGGLE_PLAY | IDLE | 暂停训练 |
| SHOW_ANSWER | NEXT_WORD | PLAYING | 下一个单词 |
| SHOW_ANSWER | KNOW | PLAYING | 确认知道 |

### 3.3 状态转移函数

**获取下一状态**:
```javascript
export function getNextState(currentState, event) {
    return stateTransitions[currentState]?.[event] || currentState;
}
```

**更新状态**:
```javascript
export function updateState(newState) {
    state.currentState = newState;
    updateStatusIndicator();
}
```

**更新状态指示器**:
```javascript
function updateStatusIndicator() {
    const indicator = document.getElementById('status-indicator');
    indicator.textContent = state.currentState;
    indicator.className = state.currentState === STATE.IDLE ? '' : 'active-status';
}
```

## 四、会话锁机制

### 4.1 设计目的

防止异步操作（如语音合成）在状态切换后继续执行，导致状态不一致。

### 4.2 核心实现

**会话熔断**:
```javascript
export function invalidateSession() {
    state.playSessionID++;
    clearAllTimers();
    window.speechSynthesis?.cancel();
}
```

**会话锁检查（在音频模块中使用）**:
```javascript
const capturedSessionID = state.playSessionID;
// 异步回调中检查:
if (state.playSessionID !== capturedSessionID) {
    resolve({ status: "aborted" }); // 熔断陈旧会话
    return;
}
```

**触发熔断的事件**:
- 切换播放状态（Space键）
- 切换速度档位（1/2/3/5键）
- 切换训练模式（M键）
- 用户输入（K/J键）

## 五、输入锁机制

### 5.1 设计目的

防止用户快速连续按键导致状态混乱。

### 5.2 使用方式

```javascript
// 在状态转移前检查
if (state.isProcessingInput && event !== 'TOGGLE_PLAY') return;

// 设置输入锁
state.isProcessingInput = true;

// 执行状态转移逻辑...

// 状态转移完成后释放锁
state.isProcessingInput = false;
```

## 六、定时器托管

### 6.1 设计目的

统一管理所有定时器，防止定时器泄漏和状态不一致。

### 6.2 安全定时器调度

```javascript
export function safeScheduleTimer(callback, delay) {
    const timerId = setTimeout(() => {
        state.activeTimersPool = state.activeTimersPool.filter(id => id !== timerId);
        callback();
    }, delay);
    state.activeTimersPool.push(timerId);
}
```

### 6.3 清除所有定时器

```javascript
export function clearAllTimers() {
    state.activeTimersPool.forEach(timerId => clearTimeout(timerId));
    state.activeTimersPool = [];
}
```

## 八、典型使用场景

### 场景1: 状态切换流程

```
1. 用户按 Space 开始训练
2. transition('TOGGLE_PLAY') 被调用
3. invalidateSession() 熔断旧会话
4. updateState(STATE.PLAYING) 更新状态
5. updateStatusIndicator() 更新UI显示
6. playNextWord() 开始播放下一个单词
```

### 场景2: 异步操作会话锁检查

```
1. speakWordAsync() 被调用
2. 捕获当前 playSessionID
3. 开始语音合成
4. 语音合成完成时检查会话ID
5. 如果会话ID已改变，中止操作
6. 如果会话ID不变，继续执行后续逻辑
```

### 场景3: 定时器托管

```
1. safeScheduleTimer(callback, 2000) 创建定时器
2. 定时器ID被加入 activeTimersPool
3. 用户按 Space 暂停训练
4. invalidateSession() 被调用
5. clearAllTimers() 清除所有定时器
6. 定时器回调不会执行
```
