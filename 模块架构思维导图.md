# 🧩 ADHD CET6 极速反应训练系统 - 模块架构思维导图 (V3)

---

## 一、整体架构概览

```
ADHD CET6 极速反应训练系统 (V3 — 三场景架构)
├── 🎨 样式模块 (CSS)          # 视觉呈现 + 三场景布局 + Combo特效
├── 📊 数据模块 (Data)          # 四个词库：全库/特高频/短语/关键词 + 常量定义
├── 🧠 状态管理模块 (State)      # FSM状态机 + 全局状态 + 按模式分离的历史栈
├── ⚙️ 核心引擎模块 (Core)       # 状态转移 + 动态加权选词 + 熟练度更新
├── ⌨️ 交互模块 (Interaction)    # 三场景键盘路由 + 词库/不熟库面板控制
├── 🔊 音频模块 (Audio)          # Web Audio API音效 + SpeechSynthesis TTS
├── 📁 场景管理                  # 训练场景 / 词库浏览场景 / 不熟词库场景
└── 💾 持久化模块 (Storage)      # localStorage + 不熟词导入导出

模块依赖关系:
键盘事件 → 交互模块(按当前场景路由) → 核心引擎 → 状态管理 → 持久化
                                          ↓            ↓
                                      音频模块      数据模块
                                          ↓            ↓
                                      视觉反馈  ←────┘
```

### 2. 📊 数据模块 (Data) — [constants.js](js/constants.js) + [wordDB.js](js/wordDB.js) + [phraseDB.js](js/phraseDB.js) + [keywordsDB.js](js/keywordsDB.js)

```
数据模块
├── 常量定义 (constants.js)
│   ├── TRAIN_MODES (SPEED_REACT, AUTO_PLAY)           # 两种训练模式
│   ├── WORD_POOLS (FULL, SUPER_HF, PHRASES, KEYWORDS) # 四个词库
│   ├── STATE (IDLE, PLAYING, WAIT_RESPONSE, SHOW_ANSWER) # 四大状态
│   ├── WORD_CATEGORIES (名词/动词/形容词/副词/介词)
│   ├── WORD_TOPICS (学术/生活/科技/经济/社会/情感/自然)

│   ├── AUDIO_CONFIG (K音/J音/释义音频率时长)
│   ├── COMBO_THRESHOLD (LOW=0, MID=5, HIGH=10, GOD=20)
│   ├── MULTI_TRANSLATION_WORDS (多重翻译单词集合)

│       lastSeenTimeStamp, nextReviewTime }

├── 翻译关键词 (keywordsDB.js)

    ├── state.recentWordsWindow → 20词环形防重队列
    └── state.unfamiliarWords → 不熟词ID列表
```

### 3. 🧠 状态管理模块 (State) — [state.js](js/state.js)

```
状态管理模块
├── 全局状态对象 state
│   ├── currentState    → IDLE | PLAYING | WAIT_RESPONSE | SHOW_ANSWER
│   ├── trainMode       → SPEED_REACT | AUTO_PLAY
│   ├── wordPool        → FULL | SUPER_HF | PHRASES | KEYWORDS
│   ├── modeSelectActive / wordPoolSelectActive → M键面板状态
│   ├── currentSpeed    → 1 | 2 | 3 | 5
│   ├── currentWord     → 当前显示的单词对象
│   ├── combo / maxCombo → 连击数
│   ├── todayCount / knowCount / unfamiliarCount → 本次统计
│   ├── historyByMode   → { SPEED_REACT: {...}, AUTO_PLAY: {...} }
│   │   └── 每个模式: { stack: [...], pointer: Number }
│   ├── playSessionID   → TTS会话锁（单调递增）
│   ├── isProcessingInput → 多键并弹防御锁
│   ├── activeTimersPool → 定时器GC托管桶
│   ├── recentWordsWindow → 20词滑动防重窗口
│   └── unfamiliarWords → 不熟词ID列表
├── 状态转移矩阵 getNextState()
│   ├── IDLE + SPACE → PLAYING
│   ├── PLAYING + SPACE → IDLE
│   ├── PLAYING + K/SPEAK_END → WAIT_RESPONSE
│   ├── PLAYING + J → SHOW_ANSWER
│   ├── WAIT_RESPONSE + K → PLAYING
│   ├── WAIT_RESPONSE + J → SHOW_ANSWER
│   └── SHOW_ANSWER + K/SPACE/NEXT_WORD → PLAYING
├── 历史栈操作
│   ├── currentHistory() → 获取当前模式的历史栈
│   ├── addToHistory(wordId) → 追加到历史栈
│   ├── goPrevWord() → 指针左移，返回wordId或null
│   └── goNextWord() → 指针右移，返回wordId或null
├── 核心函数
│   ├── updateState(newState) → 合并更新 + 刷新UI
│   ├── invalidateSession() → playSessionID++ + 清定时器 + 取消语音
│   ├── safeScheduleTimer(callback, delay) → 注册到托管桶
│   ├── clearAllTimers() → 全清桶内定时器
│   └── updateCombo(newCombo) → 更新连击 + 刷新ComboUI
```

### 4. ⚙️ 核心引擎模块 (Core) — [core.js](js/core.js)

```
核心引擎模块
├── 状态机核心
│   └── transition(event) → 输入锁检查 → 获取下一状态 → 分发handle
├── 状态处理函数
│   ├── handleIdleState() → 显示"按Space开始"，清空舞台
│   ├── handlePlayingState(event, prevState) → 核心调度
│   │   ├── prevState=WAIT_RESPONSE+event=K → 认识流程 → nextWord()
│   │   ├── prevState=SHOW_ANSWER+AUTO_PLAY → nextWord()
│   │   └── 首次进入 → 采样新词 → displayWord() → 朗读
│   ├── handleWaitResponseState() → 等待K/J
│   └── handleShowAnswerState() → 展示释义 + 加入不熟词库
├── 选词算法 getRandomWord()
│   ├── 按 wordPool 过滤词库
│   ├── BaseWeight = 11 - masteryScore
│   ├── TimeFactor 遗忘增益
│   ├── 滑动窗口防重 (FinalWeight=0)
│   └── 权重累加 + 随机落点采样
├── 单词渲染
│   ├── displayWord(word) → 显示英文+音标，隐藏释义
│   ├── showAnswer() → 展示释义+例句（含不熟词标记样式）
│   └── displayWordFromHistory(word) → 历史回看全量展示
├── 下一词调度 nextWord()
│   ├── 通过 safeScheduleTimer 延迟执行
│   ├── 推进历史栈历史栈
│   ├── 采样新词 → displayWord()
│   └── AUTO_PLAY模式: speakAndAutoShow()
├── 熟练度系统
│   ├── updateMastery(word, success) → K:+1 / J:-2
│   ├── updateCombo(count) → 更新Combo + UI
│   └── addToUnfamiliar(word) → 加入不熟列表 + 持久化
├── 模式/词库切换
│   ├── setSpeed(speed) → 熔断当前session + 更新速度
│   ├── setTrainMode(mode) → 切换训练模式
│   ├── setWordPool(pool) → 切换词库来源
│   └── skipToNext() → 跳过当前词（自动播放K键快捷操作）
└── 统计更新
    └── updateStats() → 刷新统计面板
```

### 6. 🔊 音频模块 (Audio) — [audio.js](js/audio.js)

```
音频模块
├── Web Audio API (零延迟合成音效)
│   ├── initAudioContext() → 懒加载 AudioContext
│   ├── playAudio(frequency, duration, type) → 基础振荡器
│   ├── playKnowSound() → 880Hz sine 0.15s
│   ├── playUnfamiliarSound() → 220Hz sawtooth 0.3s
│   └── playShowMeaningSound() → 440Hz sine 0.1s
├── Web Speech API (TTS语音合成)
│   └── speakWordAsync(text, lang, rate) → Promise
│       ├── capturedSessionID = state.playSessionID (闭包锁)
│       ├── 回调中检查 playSessionID 是否过期 (过期则熔断)
│       ├── utterance 挂载到 window.globalAudioGarbageCollectorBucket (防GC)
│       └── onerror → 安全解构为 resolve({status:"error"})
└── 3x/5x 速度补偿
    ├── 3x: rate锁定2.2 + gap压缩至15ms
    └── 5x: 关闭TTS，纯文字闪现代替
```
