## 一、项目定位与核心愿景

### 1.1 项目全称
**ADHD版 CET6 英语词汇瞬时反应速度训练器 (ADHD CET6 Speed Reaction Trainer - V3)**

### 1.2 核心定位变更

| 版本 | 定位 |
|------|------|
| V1 | 简单的 K/J 记忆判断工具 |
| V2 | 高频多巴胺音游式反应训练器 |
| **V3** | **三场景自适应词汇中枢**：训练场 + 词库浏览器 + 不熟词管理器，统一在键盘流下运作 |

### 1.3 设计铁律（继承自 V2）
1. **全键盘流** — 所有高频操作完全映射到键盘黄金手位
2. **零延迟反馈** — 每次按键必须有即时音效 + 视觉响应
3. **防注意力漂移** — 消除所有弹窗、模态等待、加载动画
4. **科学回流** — 永不删除单词，只降低出现概率

## 二、基础交互层（所有功能的基石）

这是系统最底层的交互协议，所有场景和模式都建立在这之上。

### 2.1 核心流程

```
[IDLE] ──Space──→ [PLAYING 显示单词] ──用户判断──→ K(认识) 或 J(不熟)
                                                      │
                    ┌─────────────────────────────────┘
                    ▼
    K → 斩杀音效 + 绿色闪烁 + Combo++ → 自动下一词
    J → 低鸣音效 + 红色提示 + Combo归零 → 展开释义 + 加入不熟词库 → 等待K继续
```

### 2.2 基础按键协议

| 按键 | 行为 | 反馈要求 |
|------|------|---------|
| **Space** | 开始 / 暂停 / 继续 | 音效 + 视觉状态切换 |
| **K** | 判定"认识" | 880Hz 清脆斩杀音 + 绿色闪烁 + Combo 递增 |
| **J** | 判定"不熟" | 220Hz 低鸣音 + 红色提示 + 自动加入不熟词库 + Combo 归零 |
| **A** / ← | 回看上一条历史 | 音效 + 展示单词及释义 |
| **D** / → | 前进到下一条 | 音效 + 若在末尾则触发新词采样 |
| **W** | 打开/关闭历史记录面板 | 面板出现/消失 + 当前词高亮跟踪 |
| **1/2/3/5** | 切换速度档位 | 音效 + 即时生效 |

### 2.3 Space 键的完整语义

Space 是三态循环中的万能钥匙：

```
在 IDLE 时按 Space  →  开始训练，抽取并显示第一个单词
在 PLAYING 时按 Space →  暂停，终止所有语音和定时器，回到 IDLE
在 SHOW_ANSWER 时按 Space →  解除释义挂起，进入下一词
```

每次 Space 按下都伴随：语音合成终止 + 定时器全清 + 音效反馈 + UI 状态切换。

### 2.4 单词随机出现机制

单词**不是**顺序出现，而是按自适应权重算法随机采样：

- 熟练度越低 → 出现概率越高
- 遗忘越久 → 出现概率越高  
- 最近出现过的词 → 被排除（20词滑动窗口防重）

（详见第六章算法层）

### 场景三：⚠ 不熟词库（U 键进入）

**用途：** 管理所有被标记为不熟悉的单词，**持久化存储**，重启不丢失。

- 支持导出/导入 JSON 文件
- 可在不熟词库内直接启动不熟词专项训练（按 L 键）
- 每个单词有独立的「移除」按钮和键盘移除操作

**进入方式：** 在任意场景按 **U** 键

**布局：**

├──────────────────────────────────┤
│  ┌────────────────┐              │
│  │▶abandon         │ [移除]      │  ← 不熟词卡 + 移除按钮
│  │/əˈbændən/      │              │
│  │vt. 放弃，遗弃    │              │

| 按键 | 行为 | 反馈 |
|------|------|------|
| **WASD** | 同词库场景，网格移动选框 | 选框移动 |
| **J** | 将当前选中词从「不熟词库」中移除 | 反馈音效 + 卡片消失动画 |
| **L** | 直接启动不熟词专项训练 | 切换到训练场景，仅用不熟词库 |
| **Esc** / **U** | 关闭回到训练 | — |
| **鼠标点击** | 点击词卡选中 / 点击[移除]按钮 | 选框跟随 / 移除确认 |

### 场景切换规则

1. 从训练场景进入 L/U 场景时，训练**暂停**（进入 IDLE）
2. 从 L/U 返回训练时，训练保持暂停状态，用户按 Space 继续
3. L 和 U 场景互斥——打开一个会自动关闭另一个
4. 在 L 或 U 场景中调整速度档位（1/2/3/5）同样生效，回到训练后立即应用

## 四、M 键模式切换系统

M 键打开一个**半透明的模式选择覆盖层**，用于切换训练模式和词库来源。

### 4.1 M 键按下后的流程

```
按 M ──→ 弹出模式选择面板，训练暂停 ──→ 用户按以下键之一：

    按 Space  →  切换为「自动播放模式」+ 用当前词库开始训练
    按 K      →  切换为「手动反应模式」+ 用当前词库开始训练
    按 U      →  切换为「手动反应模式」+ 用不熟词库开始训练
    按 W      →  进入「词库选择模式」（A/D 切换词库）
    按 M/ESC  →  关闭面板，回到之前的状态
```

### 4.2 两种训练模式

| 模式名 | 触发键 | 行为 |
|--------|--------|------|
| **手动反应** `SPEED_REACT` | K (在M面板中) | 显示单词 → 等待用户按 K/J → 反馈 → 下一词 |
| **自动播放** `AUTO_PLAY` | Space (在M面板中) | 自动读单词 → 停顿 → 显示释义 → 停顿 → 自动下一词 |

### 4.3 词库选择（M → W → A/D）

```
M → W 进入词库选择 →
    A/D 循环切换:
        [全库] ⇄ [特高频词] ⇄ [短语] ⇄ [翻译关键词]
    W 确定并返回
```

| 词库 | 常量 | 内容 |
|------|------|------|
| 全库 | `FULL` | 全部 CET6 词汇 |
| 特高频 | `SUPER_HF` | 50 个最高频词 |
| 短语 | `PHRASES` | CET6 高频短语 |
| 关键词 | `KEYWORDS` | 翻译关键词汇 |

**不熟词库作为词源：** 在 M 面板中按 U 即可用不熟词库进行训练，独立于上述四个标准词库。

### 4.4 模式 / 词库切换的竞态安全

每次模式切换必须执行：
1. `state.playSessionID++` — 会话锁递增
2. `clearAllTimers()` — 清空全部定时器
3. `speechSynthesis.cancel()` — 终止所有语音（执行两次，防范 Chrome bug）
4. `state.isProcessingInput = false` — 释放输入锁

## 六、数据架构与选词算法

### 6.1 单词数据结构

```javascript
{
  id: Number,          // 唯一标识
  word: "abandon",     // 英文
  phonetic: "/əˈbændən/", // 音标
  meaning: "vt. 放弃",  // 中文释义
  example: "The heavy snow forced them to abandon their car.", // 例句
  category: "动词",     // 词性分类
  topic: "生活",        // 主题分类
  level: "基础",        // 难度等级

  // === 算法控制指标 ===
  masteryScore: 0,     // 熟练度 [-10, 10]，初始 0
  knownCount: 0,       // 累计认识次数
  unknownCount: 0,     // 累计不认识次数
  appearCount: 0,      // 累计出现次数
  lastSeenTimeStamp: 0,  // 上次出现时间
  nextReviewTime: 0      // 建议下次复习时间
}
```

### 6.2 动态权重采样算法

**Step 1 — 基础权重：**
```
BaseWeight = 11 - masteryScore    (masteryScore ∈ [-10, 10])
```
- 熟练度 -10（极不熟）→ 权重 21（最高概率）
- 熟练度 0（中性）→ 权重 11
- 熟练度 10（完全掌握）→ 权重 1（最低概率，但永不归零）

**Step 2 — 遗忘催化增益：**
```
TimeFactor = 1 + (当前时间 - lastSeenTimeStamp) / 遗忘衰减跨度
FinalWeight = BaseWeight × TimeFactor
```
长时间未见的"熟练词"会被遗忘催化拉高权重，强制回流复习。

**Step 3 — 滑动窗口防重（20 词环形队列）：**
```
if (wordIndex in state.recentWordsWindow) {
    FinalWeight = 0  // 近期出现的词被彻底排除
}
```

### 6.3 K/J 按键状态回流矩阵

| 操作 | masteryScore 变化 | Combo | 复现调度 |
|------|-------------------|-------|---------|
| 按 K（认识） | +1（上限10） | +1 | 当前时间 + masteryScore × 30分钟 |
| 按 J（不熟） | -2（下限-10） | 归零 | 当前时间 + 90秒（紧急回流） |

## 七、FSM 状态机架构

### 7.1 四大顶层状态

```
     ┌────────────── IDLE (空闲) ──────────────┐
     │    Space → 启动训练                      │
     ▼                                          │
  PLAYING (播放中) ─────── Space → 暂停 ────────┘
     │                        │
     │ SPEAK_END 或 K         │ J
     ▼                        ▼
  WAIT_RESPONSE (等待响应)  SHOW_ANSWER (显示释义)
     │   K → 下一词             │
     │   J → SHOW_ANSWER        │
     │                          │
     └──────── K/Space → 下一词 ←┘
```

### 7.2 状态转移表

| 当前状态 | 事件 | 目标状态 | 必须执行的清理动作 |
|---------|------|---------|-------------------|
| IDLE | Space | PLAYING | 生成新 playSessionID |
| PLAYING | Space | IDLE | invalidateSession() → 清定时器 + 取消语音 |
| PLAYING | K / SPEAK_END | WAIT_RESPONSE | — |
| PLAYING | J | SHOW_ANSWER | 展开释义面板 |
| WAIT_RESPONSE | K | PLAYING | Combo++，更新 masteryScore，调用 nextWord() |
| WAIT_RESPONSE | J | SHOW_ANSWER | Combo 归零，展开释义 |
| SHOW_ANSWER | K / Space / NEXT_WORD | PLAYING | 步进下一词 |

### 7.3 场景切换时的状态处理

场景切换（训练 ⇄ 词库 ⇄ 不熟库）时：
- 如果当前在 PLAYING 或 WAIT_RESPONSE 状态，先执行 `invalidateSession()`
- 将训练状态强制设为 IDLE
- 进入新场景后，键盘事件路由到对应场景的处理函数

## 八、技术防御层

### 8.1 playSessionID 异步会话锁

```javascript
// 每次破坏当前播放流的操作之前：
state.playSessionID++;

// 所有异步回调内部：
const capturedSessionID = state.playSessionID;
utterance.onend = () => {
    if (state.playSessionID !== capturedSessionID) {
        return; // 会话已过期，熔断
    }
    // 正常处理...
};
```

### 8.2 定时器 GC 托管桶

```
禁止使用孤立的 setTimeout！
所有定时器必须通过 safeScheduleTimer() 创建并注册到 state.activeTimersPool。
任何状态切换必须先调用 clearAllTimers() 清空旧定时器。
```

### 8.3 多键并弹输入锁

```javascript
state.isProcessingInput = false;

// 键盘事件入口：
if (state.isProcessingInput && event !== 'SPACE') return;
state.isProcessingInput = true;

// 单词渲染完成后：
state.isProcessingInput = false;
```

### 8.4 浏览器 TTS 补偿策略

| 档位 | rate 设置 | 策略 |
|------|-----------|------|
| 1x | 0.8 | 标准朗读 |
| 2x | 1.5 | 快语速 |
| 3x | **2.2**（封顶）+ 间隔压缩至 15ms | 绕开 Chrome 2.5x 物理极限 |
| 5x | 不调用 TTS | 纯文字闪现 + 极短定时器 |

## 十、数据持久化策略

### 10.1 存储键

| 键 | 内容 | 频率 |
|----|------|------|
| `adhd-word-progress` | 全部单词的 masteryScore 等状态 | 3秒节流写入 |
| `adhd-unfamiliar-words` | 不熟词 ID 列表 | 变化时即时写入 |

### 10.2 不熟词库的生命周期

```
启动 → 从 localStorage 加载不熟词列表
训练中 → 按 J = 自动加入 + 即时写入 localStorage
词库浏览 → 按 J = 手动标记 + 即时写入
不熟词库 → 按 J = 手动移除 + 即时写入
导出 → 下载为 .json 文件
导入 → 从 .json 合并到当前列表（去重）
```

不熟词库**永不自动清空**，除非用户手动移除或清除浏览器数据。
