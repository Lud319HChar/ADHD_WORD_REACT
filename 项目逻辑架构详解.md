## 一、从零理解：这个项目是干什么的

### 需求的起点

你要做一个 **CET6 单词速刷工具**，专门适配 ADHD 用户的注意力特点：
- 单词不是顺序背，而是 **随机出现**，越不熟的词出现频率越高
- 每次操作都有 **即时音效 + 视觉反馈**（多巴胺反馈）
- 全键盘操作，不需要鼠标

### 从需求文档看演化

| 版本 | 核心概念 | 文件状态 |
|------|----------|----------|
| V2 | 单一训练页，M键切4种模式 | [NewADHDWordV2.html](file:///h%3A/Note/HTMLtest/NewADHDWordV2.html)（旧版） |
| V3 | 引入「三场景」架构：训练 / 词库 / 不熟库 | [需求ADHD六级词汇反应训练.md](file:///h%3A/Note/HTMLtest/需求ADHD六级词汇反应训练.md) |
| V4 | 细化为模块化架构 + 中枢面板 + iframe 训练页 | [需求_V4_系统设计文档.md](file:///h%3A/Note/HTMLtest/需求_V4_系统设计文档.md) |

### 最终形态（V4）

你当前的项目是一个 **中枢 + 内嵌训练页** 的结构：

```
hube.html（中枢窗口）
├── Tab 1: 极速训练 → iframe 嵌入 index.html（训练核心）
├── Tab 2: 完整词库 → 947 个 CET6 单词的卡片网格
├── Tab 3: 特高频词 → 50 个最高频词
├── Tab 4: 高频短语 → 150 条短语
├── Tab 5: 翻译关键词 → 按主题分类的词汇群组
└── Tab 6: 全局搜索 → 跨词库搜索
```

## 三、从用户按键到屏幕变化：一条完整的调用链

> 追踪「用户按 K 知道一个单词」的全过程，理解所有函数是如何联动的。

### 步骤 1：键盘按下 → interaction.js

```
用户按下 K 键
    │
    ▼
index.html → <script type="module">
    │  document.addEventListener('keydown', handleKeyDown)
    ▼
interaction.js → handleKeyDown(e)
    │
    ├── 1. 检查：词库弹窗开着吗？→ 不是
    ├── 2. 检查：不熟弹窗开着吗？→ 不是
    ├── 3. 检查：M 面板开着吗？→ 不是
    └── 4. 进入训练场景 switch → case 'K': handleK()
```

### 步骤 2：handleK() → transition('K')

```
interaction.js → handleK()
    │
    ├── 如果是 AUTO_PLAY 模式（K 当"跳过"用）
    │   └── invalidateSession() → import('./core.js').then(m => m.skipToNext())
    │
    └── 如果是手动模式（K 当"认识"用）
        └── transition('K')
```

### 步骤 3：transition() 状态机核心

```
core.js → transition('K')
    │
    ├── 输入锁检查：isProcessingInput? → 锁定中则 return（防狂按）
    │
    ├── getNextState('WAIT_RESPONSE', 'K')
    │   → 查转移矩阵 → 下一状态 = 'PLAYING'
    │
    ├── invalidateSession()
    │   → state.js → playSessionID++           ← TTS 会话锁递增（旧的朗读被熔断）
    │   → state.js → clearAllTimers()          ← 清空所有定时器
    │   → speechSynthesis.cancel()              ← 停止当前朗读
    │
    └── switch(nextState = 'PLAYING')
        └── handlePlayingState('K', 'WAIT_RESPONSE')
```

### 步骤 4：handlePlayingState 执行"认识"逻辑

```
core.js → handlePlayingState('K', 'WAIT_RESPONSE')
    │
    ├── 分支判断：prevState === WAIT_RESPONSE → 是"认识"流程
    │
    ├── playKnowSound()               ← audio.js → 880Hz sine 0.15s 斩杀音
    │
    ├── updateMastery(currentWord, true)   ← 熟练度 +1
    │   │
    │   ├── wordDB 中找到对应词条
    │   ├── masteryScore = min(10, score+1)       ← 上限 10
    │   ├── nextReviewTime = now + score×30分钟    ← 越熟回流越慢
    │   ├── appearCount++ / lastSeenTimeStamp = now
    │   ├── state.knowCount++                     ← 今日统计 +1
    │   └── saveProgress()                        ← storage.js → 3 秒节流写入 localStorage
    │
    ├── updateCombo(state.combo + 1)      ← state.js → Combo 显示 + 视觉等级变色
    │
    └── nextWord()                        ← 抽取下一个词
        │
        └── safeScheduleTimer(
              getRandomWord → displayWord → speakWordAsync,
              delay
            )
```

### 步骤 5：下一个词出现

```
core.js → nextWord()
    │
    └── safeScheduleTimer(() => {
          │
          ├── getRandomWord()           ← 三层权重算法选词
          │   ├── 1. 按 wordPool 过滤（全库/特高频/短语/关键词）
          │   ├── 2. 排除 recentWordsWindow（20 词防重）
          │   ├── 3. 排除 nextReviewTime 未到的
          │   └── 3. BaseWeight + TimeFactor 加权随机
          │
          ├── updateState({ currentWord })   ← state.js → 更新当前词
          │
          ├── displayWord(newWord)           ← DOM 操作
          │   ├── #word-display.textContent = word.word       ← 6rem 大字
          │   ├── #phonetic-display.textContent = phonetic    ← 灰色音标
          │   └── 释义/例句 → 移除 .show-meaning → 隐藏
          │
          ├── addToHistory(newWord.id)       ← state.js → 写入 per-mode 历史栈
          │
          └── 朗读（按模式/速度）
              ├── AUTO_PLAY → speakAndAutoShow()
              ├── speed ≠ 5 → speakWordAsync().then(transition('SPEAK_END'))
              └── speed = 5 → safeScheduleTimer(transition('SPEAK_END'), gap)
        })
```

## 四、逐模块详解：每个 JS 文件干了什么

### 4.1 constants.js — 常量字典

> **角色**：被所有模块引用的配置中心，没有逻辑，只有数据。

```javascript
// 所有模块都 import 这个文件来取枚举值

TRAIN_MODES    → 两种模式：SPEED_REACT（手动反应）、AUTO_PLAY（自动播放）
WORD_POOLS     → 四种词库：FULL（全库）、SUPER_HF（特高频）、PHRASES（短语）、KEYWORDS（关键词）
STATE          → 四种状态：IDLE、PLAYING、WAIT_RESPONSE、SHOW_ANSWER
SPEED_CONFIG   → 四种速度：1x(rate:0.8, gap:2000ms)、2x(1.5,1500)、3x(2.2,1000)、5x(3.0,500)
AUDIO_CONFIG   → 三种音效：K(880Hz 0.15s)、J(220Hz 0.3s)、显示释义(440Hz 0.1s)
COMBO_THRESHOLD→ 四级连击：LOW(0)、MID(5)、HIGH(10)、GOD(20)
SUPER_HF_WORDS → 50 个特高频词的 Set 集合
WORD_CATEGORIES→ 词性枚举（名词/动词/形容词/副词/介词）
WORD_TOPICS    → 主题枚举（学术/生活/科技/经济/社会/情感/自然）
WORD_LEVELS    → 难度枚举（基础/进阶/高级）
```

### 4.2 state.js — 大脑中枢

> **角色**：持有整个应用的全局状态对象 `state`，所有模块都读写它。

**核心职责**：

```
state 对象（约 30 个字段，见下方）
    │
    ├── 系统状态
    │   ├── currentState     → 当前 FSM 状态
    │   ├── trainMode        → 当前训练模式
    │   ├── wordPool         → 当前词库来源
    │   ├── currentSpeed     → 当前速度档位
    │   └── modeSelectActive / wordPoolSelectActive → M 面板状态
    │
    ├── 训练上下文
    │   ├── currentWord      → 当前显示的单词对象
    │   ├── todayCount / knowCount / unfamiliarCount → 今日统计
    │   └── combo / maxCombo → 连击系统
    │
    ├── 技术防御
    │   ├── playSessionID    → TTS 会话锁（每次切词 +1，旧朗读被熔断）
    │   ├── isProcessingInput→ 输入锁（防狂按 K/J）
    │   └── activeTimersPool → 定时器桶（随时可全清）
    │
    ├── 数据
    │   ├── unfamiliarWords  → 不熟词 ID 列表 [1, 5, 23, ...]
    │   └── recentWordsWindow→ 20 词防重环形队列
    │
    └── 历史栈（per-mode，每个模式独立 16 条记录）
        └── historyByMode = {
              speed_react: { stack: [12,45,388,21], pointer: 2 },
              auto_play:   { stack: [5,8,15],     pointer: 1 },
              fill_blank:  { stack: [],            pointer: -1 }
            }
```

**导出的函数**：

| 函数 | 调用者 | 作用 |
|------|--------|------|
| `getNextState(current, event)` | core.js | 查转移矩阵，返回下一状态 |
| `updateState({...})` | core.js / interaction.js | 合并更新 state + 自动刷新 UI 栏 |
| `invalidateSession()` | core.js / interaction.js | playSessionID++ + 清定时器 + 取消 TTS |
| `safeScheduleTimer(fn, ms)` | core.js | 创建定时器并注册到 activeTimersPool |
| `clearAllTimers()` | core.js | 全清托管桶 |
| `updateCombo(n)` | core.js | 更新连击 + 自动刷 DOM 和颜色 |
| `currentHistory()` | interaction.js | 获取当前模式的历史栈 |
| `addToHistory(id)` | core.js | 追加到历史栈 |
| `goPrevWord()` | interaction.js | A 键 → 历史指针 -1 |
| `goNextWord()` | interaction.js | D 键 → 历史指针 +1 |

### 4.3 core.js — 核心引擎

> **角色**：所有训练逻辑的调度中心，是项目中函数调用链最长、依赖最多的模块。

**导出的函数**：

| 函数 | 调用方 | 作用 |
|------|--------|------|
| `transition(event)` | interaction.js | **状态机入口**，根据当前状态和事件调用对应 handler |
| `getRandomWord()` | core.js 内部 | 三层加权随机选词算法 |
| `displayWord(word)` | core.js 内部 | 渲染单词英文 + 音标到 DOM |
| `showAnswer()` | core.js 内部 | 渲染释义 + 例句到 DOM（淡入动画） |
| `addToUnfamiliar(word)` | interaction.js / core.js | 加入不熟词库 |
| `removeFromUnfamiliar(id)` | interaction.js | 移除不熟词 |
| `skipToNext()` | interaction.js | 自动播放中 K 键打断，跳到下一词 |
| `setSpeed(n)` | interaction.js | 更新速度 + 熔断当前会话 |
| `setTrainMode(mode)` | interaction.js | 切换训练模式 + 熔断 |
| `setWordPool(pool)` | interaction.js | 切换词库（下次选词生效） |

**内部 handler（不导出，仅 transition 调用）**：

```
handleIdleState()
    → 清空舞台，显示"按 SPACE 开始"

handlePlayingState(event, prevState)
    ├── 若 prevState = WAIT_RESPONSE → "认识"流程（K 后下一词）
    ├── 若 AUTO_PLAY + prevState = SHOW_ANSWER → 自动下一词
    └── 否则 → 首次进入，采样新词 + 朗读

handleWaitResponseState(event)
    → showAnswer() + playShowMeaningSound()

handleShowAnswerState(event)
    ├── 若 event = 'J' → 不熟音 + 熟练度 -2 + Combo 归零
    └── 自动定时器 → transition('NEXT_WORD') 超时后下一词
```

**选词算法 `getRandomWord()` 的完整流程**：

```
1. 按 wordPool 过滤词库
   ├── FULL      → wordDB 全部 (947 词)
   ├── SUPER_HF  → wordDB ∩ SUPER_HF_WORDS (约 50 词)
   ├── PHRASES   → phraseDB 映射为临时 word 对象
   └── KEYWORDS  → keywordsDB 所有群组的 wordIds → wordDB 子集

2. 可用词过滤
   ├── 排除 recentWordsWindow 中的词（20 词防重）
   └── 排除 nextReviewTime 未到的词

3. 加权随机采样
   对每个可用词:
     BaseWeight = max(1, 11 - masteryScore)     ← 不熟 → 权重高
     TimeFactor = 1 + (now-lastSeen) / 24小时    ← 久未见 → 权重高
     最终权重 = BaseWeight × TimeFactor
   → 汇总总权重 → Math.random() × totalWeight → 按权重落点选中

4. 将选中词 ID 推入 recentWordsWindow
```

### 4.4 interaction.js — 键盘路由器

> **角色**：全局唯一的键盘事件入口，按当前场景分发到不同的处理函数。

```
handleKeyDown(e)                         ← 唯一入口（所有按键都经过这里）
    │
    ├── 优先级 1：isWordLibraryOpen()?    → handleWordLibraryKeys()
    │        L 弹窗开着 → WASD/Q/E/J/Esc 都在这里处理
    │
    ├── 优先级 2：isUnfamiliarLibraryOpen()? → handleUnfamiliarLibraryKeys()
    │        U 弹窗开着 → WASD/J/L/Esc 都在这里处理
    │
    ├── 优先级 3：state.modeSelectActive? → handleModeSelectKeys()
    │        M 面板开着 → Space/K/W/U 都在这里处理
    │
    ├── 优先级 4：state.wordPoolSelectActive? → handleWordPoolSelectKeys()
    │        词库选择子面板开着 → A/D/W 都在这里处理
    │
    └── 优先级 5：训练场景（默认）
        switch(key):
          Space → handleSpace()
          K     → handleK()
          J     → handleJ()
          A/←   → handleA()
          D/→   → handleD()
          W     → toggleHistoryPanel()
          L     → openWordLibrary()
          U     → openUnfamiliarLibrary()
          M     → toggleModeSelect()
          1/2/3/5 → setSpeed()
```

**场景切换规则**：

```
训练 → L/U 词库：
  1. invalidateSession() → 暂停训练
  2. currentState = IDLE
  3. 打开弹窗

L 和 U 互斥：
  打开 L → 自动关闭 U
  打开 U → 自动关闭 L

L/U → 训练：
  1. 关闭弹窗
  2. 保持 IDLE，用户按 Space 重新开始
```

### 4.5 audio.js — 声音系统

> **角色**：两种技术合成所有声音，零音频文件加载。

**Web Audio API（实时合成音效）**：
```
playAudio(freq, dur, type)
    → OscillatorNode（振荡器）→ GainNode（增益）→ 扬声器
    → 0.3 音量 → 指数衰减到 0.01（自然淡出）

playKnowSound()          → 880Hz sine 0.15s  ← K 键清脆斩杀
playUnfamiliarSound()    → 220Hz sawtooth 0.3s ← J 键低沉挫败
playShowMeaningSound()   → 440Hz sine 0.1s   ← 显示释义
```

**Web Speech API（TTS 朗读）**：
```
speakWordAsync(text, lang, rate)
    → return new Promise
    → 闭包捕获 capturedSessionID = state.playSessionID
    → onend 回调：
        if (当前 sessionID !== capturedSessionID) → resolve({status:"aborted"})  ← 熔断
        else → resolve({status:"success"})
    → onerror → resolve({status:"error"})  ← 不抛异常，防止卡死
    → 全局挂载防 GC：window.globalAudioGarbageCollectorBucket = utterance
```

### 4.6 storage.js — 数据持久化

> **角色**：把训练进度和不熟词存到浏览器 localStorage。

```
两个存储键：
  'adhd-word-progress'    → 947 个词各的 masteryScore、appearCount、时间戳
  'adhd-unfamiliar-words' → 不熟词 ID 数组

写入策略：
  saveProgress()          → 3 秒节流（高频 K/J 不阻塞 UI）
  saveUnfamiliarWords()   → 即时写入（加入/移除立即落盘）

启动恢复：
  loadProgress()          → JSON.parse → 逐个回填 wordDB 中对应词的字段
  loadUnfamiliarWords()   → JSON.parse → state.unfamiliarWords + 同步 count

不熟词导入导出：
  downloadUnfamiliarWords()→ JSON.stringify → Blob → <a> 下载 .json
  importUnfamiliarWords()  → FileReader → JSON.parse → 合并去重
```

### 4.7 数据模块 — wordDB / phraseDB / keywordsDB

```
wordDB.js (947 条)
  └── 每条：{ id, word, phonetic, meaning, example, category, topic, level,
              masteryScore(运行时), appearCount(运行时), lastSeenTimeStamp(运行时),
              nextReviewTime(运行时) }

phraseDB.js (150 条)
  └── 每条：{ id, phrase, meaning, example, category }
  └── 训练时被 getRandomWord() 映射为临时 word 对象 (id = i+10000)

keywordsDB.js (8 群组)
  └── 每条：{ keyword, english, description, wordIds: [...] }
  └── 训练时所有 wordIds 合并去重 → wordDB 子集
```

### 4.8 search.js — 全局搜索

```
searchWords(query)      → wordDB 中 word/phonetic/meaning/example 模糊匹配 → 前 50 条
searchPhrases(query)    → phraseDB 中 phrase/meaning/example 匹配 → 前 30 条
searchKeywords(query)   → keywordsDB 中 keyword/english/description 匹配 → 前 10 条
getSuggestions(query)   → 实时自动建议（前 5 单词 + 前 3 短语）
renderSearchResults()   → 渲染结果到 DOM
```

## 六、server.js：手机仪表盘是怎么联动的

> [mobile-dashboard.html](file:///h%3A/Note/HTMLtest/mobile-dashboard.html) 和 [server.js](file:///h%3A/Note/HTMLtest/server.js) 构成了「手机监控电脑训练」的功能。

```
┌──────────────────┐          HTTP           ┌──────────────────┐
│  手机浏览器       │ ←───────→ │ 电脑 node server.js │
│  mobile-dashboard │  局域网    │ (和 index.html 同一个）│
│  .html            │  WiFi      │                   │
└────────┬─────────┘            └────────┬──────────┘
         │                              │
         │  GET /api/stats (每5秒轮询)    │ 读 server-progress.json
         │ ← 返回训练进度 JSON            │
         │                              │
         │  POST /api/command            │ 写 commands.json
         │  发送指令 (如"切换到2X")       │
         │                              │
         └──────────────────────────────┘
```

**数据流**：

```
电脑侧 index.html 训练时：
    → localStorage.setItem('adhd-word-progress', ...)
    → 还没同步到 server-progress.json

如果需要手机仪表盘看到实时数据：
    需要在训练代码中额外调用 POST /api/progress，
    把当前训练状态写入 server-progress.json

手机仪表盘：
    每 5 秒 GET /api/stats → server.js 读 server-progress.json → 返回
    用户按"发送" → POST /api/command → 写入 commands.json
```

### server.js 的 API 端点一览

| 端点 | 方法 | 作用 | 风险等级 |
|------|------|------|---------|
| `/` 及其他路径 | GET | 返回静态文件（HTML/CSS/JS） | 🟢 |
| `/api/progress` | GET | 读取训练进度 | 🟡 无认证 |
| `/api/progress` | POST | 写入训练进度（合并策略） | 🔴 无认证 |
| `/api/stats` | GET | 返回仪表盘统计数据 | 🟡 无认证 |
| `/api/command` | GET | 读取待执行命令（手机→电脑） | 🟡 无认证 |
| `/api/command` | POST | 发送命令 | 🔴 无认证 |
| `/api/command/ack` | POST | 确认命令已读取 | 🟡 无认证 |

## 总结：一句话概括每个模块的角色

```
hube.html      = 中枢窗口，6 个 Tab，负责词库浏览和全局搜索
index.html     = 训练核心（iframe 嵌入），加载全部 JS 引擎
constants.js   = 字典（所有枚举值和配置常量）
state.js       = 大脑（全局状态 + FSM + 历史栈 + 技术防御锁）
core.js        = 心脏（transition 状态机 + 选词算法 + 渲染 + 熟练度）
interaction.js = 路由器（全局键盘事件 → 按场景分发给不同处理函数）
audio.js       = 声音（Web Audio 合成音效 + TTS 朗读 + 竞态锁）
storage.js     = 存储（localStorage 读写 + 节流 + 导入导出）
search.js      = 搜索（跨词库模糊匹配）
wordDB.js      = 词库数据（947 个 CET6 单词）
phraseDB.js    = 短语数据（150 条）
keywordsDB.js  = 关键词数据（8 个语义群组）
server.js      = 服务端（静态文件 + API，供手机仪表盘通信）
```
