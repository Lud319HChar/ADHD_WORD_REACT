## 一、系统概述与核心定位

### 1.1 项目名称

**ADHD CET6 词汇极速反应训练中枢 V4**

### 1.2 核心定位

三场景自适应词汇中枢：**训练场 → 词库浏览器 → 不熟词管理器**，全部在全键盘流 + 即时音效/视觉反馈下运作。

### 1.3 用户核心逻辑（自下而上）

```
基础层：K/J 熟悉/不熟速判 → Space 启/停/续 → 每次操作都有声音+视觉反馈
    ↓
浏览层：A/D 前后浏览历史 → 单词按算法随机出现 → 永不重复刚出现的词
    ↓
子面板：W 历史小面板（当前词追踪 + 鼠标可点击）
    ↓
三个场景：
  场景一 = 训练主界面（默认）
  场景二 = L 词库浏览（WASD 选框 + J 标不熟 + 音效+视觉提示"已加入"）
  场景三 = U 不熟词库（持久化存储 + 导入导出 + 可直接启动不熟词训练）
    ↓
模式层：M 键
  → Space = 自动播放模式
  → K     = 手动反应模式
  → U     = 不熟词库训练
  → W     = 词库选择（A/D 在 全库/特高频/短语/关键词 间切换）
```

## 二、基础交互层

### 2.1 核心操作流程

```
[IDLE 空闲]
    │ 按 Space → 有音效(330Hz sine) + 状态变为 PLAYING
    ▼
[PLAYING 显示单词]
    │ 朗读英文（TTS，速度由当前档位决定）
    │
    ├── 用户按 K → 认识
    │   └── 880Hz sine 清脆斩杀音(0.15s)
    │       + Combo 数值递增 + UI 变色
    │       + 释义面板淡入
    │       + 熟练度 +1（上限10）
    │       + 自动跳下一词
    │
    ├── 用户按 J → 不熟
    │   └── 220Hz sawtooth 低鸣挫败音(0.3s)
    │       + Combo 归零 + 特效全消失
    │       + 释义面板淡入 + 红色脉冲动画
    │       + 自动加入不熟词库
    │       + 熟练度 -2（下限-10）
    │       + 屏幕显示释义 + 等待 K 继续
    │
    └── 用户按 Space → 暂停
        └── 330Hz sine 音效(0.05s)
            + 状态变 IDLE
            + 语音合成终止 + 定时器全清
            + 屏幕显示"按 SPACE 开始"
```

### 2.2 完整基础按键表

| 按键 | 功能 | 音效反馈 | 视觉反馈 | 状态要求 |
|------|------|---------|---------|---------|
| **Space** | 开始/暂停/继续 | 330Hz sine 0.05s | 状态切换：IDLE↔PLAYING | IDLE / PLAYING / SHOW_ANSWER |
| **K** | 认识 | 880Hz sine 0.15s（清脆斩杀） | Combo++ / 释义淡入 | PLAYING / WAIT_RESPONSE |
| **J** | 不熟 | 220Hz sawtooth 0.3s（低沉挫败） | Combo归零 / 红色不熟脉冲动画 / 自动加入不熟库 | PLAYING / WAIT_RESPONSE / SHOW_ANSWER |
| **A / ←** | 回看上一条 | 无额外音效 | 显示单词 + 释义（历史态） | 全部 |
| **D / →** | 前进/新词 | 无额外音效 | 显示单词 + 释义 / 触发新词采样 | 全部 |
| **W** | 历史小面板 | 无额外音效 | 底部滑入/滑出 + 当前词高亮追踪 | 全部（训练场景内） |
| **1/2/3/5** | 调速 | 无额外音效 | 状态栏即时更新速度档位 | 全部 |

### 2.3 J 键的特殊语义（双用途）

```
在 PLAYING / WAIT_RESPONSE 状态：
  按 J → transition('J') → SHOW_ANSWER → 展开释义 + 加入不熟库 + 红色脉冲

在 SHOW_ANSWER 状态（A/D 回退到历史词后）：
  按 J → addToUnfamiliar(word) + 即时保存 + 红色脉冲 → transition('SPACE')
```

SHOW_ANSWER 分支的存在原因是：用户回退到历史词后也想标记不熟时，`transition('J')` 会把当前历史词覆盖掉，所以直接在 handleJ 中走新增的 SHOW_ANSWER 分支直接标记 + 持久化。

## 四、语音提示系统（全覆盖）

### 4.1 Web Audio API 音效表

所有音效使用浏览器原生 Web Audio API 的 OscillatorNode 实时合成，**零延迟，无需加载音频文件**。

| 事件 | 频率 | 波形 | 时长 | 音量曲线 | 触发时机 |
|------|------|------|------|---------|---------|
| **K 键（认识）** | 880Hz | sine | 0.15s | 0.3 → exponential → 0.01 | 用户按下 K |
| **J 键（不熟）** | 220Hz | sawtooth | 0.3s | 0.3 → exponential → 0.01 | 用户按下 J |
| **显示释义** | 440Hz | sine | 0.1s | 0.3 → exponential → 0.01 | 释义面板展开时 |
| **Space（启停）** | 330Hz | sine | 0.05s | 0.3 → exponential → 0.01 | Space 状态切换时 |

**指数衰减曲线**：`gainNode.gain.exponentialRampToValueAtTime(0.01, ctx.currentTime + duration)`，确保音效自然淡出而非突兀切断。

### 4.2 Web Speech API TTS 朗读

| 档位 | rate | 中文间隔 gap | 行为 |
|------|------|-------------|------|
| **1x** | 0.8 | 2000ms | 标准朗读 |
| **2x** | 1.5 | 1500ms | 快语速 |
| **3x** | 2.2（封顶） | 1000ms | 绕过 Chrome 2.5x 物理极限 |
| **5x** | 不调用 TTS | 500ms | 纯文字闪现模式 |

**会话锁保护**（详见第十一章）：
- 每次调用 `speakWordAsync` 前捕获 `state.playSessionID`
- `onend` 回调中检查：如果 ID 已变（用户切词/暂停）→ 熔断，返回 `{status:"aborted"}`
- `onerror` 安全解构为 `{status:"error"}`，绝不抛异常卡死状态机

### 4.3 AudioContext 懒加载

```javascript
// 首次播放音效时才创建 AudioContext
// 避免页面加载时不必要的资源消耗
// 兼容 webkitAudioContext（旧版 Safari）
```

1. 进入 L/U 场景 → 训练**暂停**（进入 IDLE + invalidateSession）

3. L 和 U 互斥：打开一个自动关闭另一个
4. 跨场景调速（1/2/3/5）即时生效，回到训练后应用

## 七、M 键模式系统

### 7.1 M 键完整流程

```
按 M →
   状态栏变为 "MODE_SEL | Space=自动播放 | W=选词库 | K=极速反应 | U=不熟训练"

   用户按键：
   ├── Space  → trainMode = AUTO_PLAY
   │            + 状态变 PLAYING
   │            + 开始自动播放轮播
   │
   ├── K      → trainMode = SPEED_REACT
   │            + 状态变 IDLE（等待用户按 Space）
   │
   ├── U      → trainMode = SPEED_REACT
   │            + 使用不熟词库作为训练来源
   │            + 状态变 IDLE
   │
   ├── W      → 进入词库选择子面板
   │            A/D = 循环切换（全库 ⇄ 特高频 ⇄ 短语 ⇄ 关键词）
```

## 九、动态加权选词算法

### 9.1 单词数据结构（`wordDB.js` 中每一项）

```javascript
{
    id: 1,
    word: "abandon",
    phonetic: "/əˈbændən/",
    meaning: "vt. 放弃，遗弃，抛弃",
    example: "The heavy snow forced them to abandon their car.",
    category: "动词",
    topic: "生活",
    level: "进阶",
    masteryScore: 0,          // 熟练度 [-10, 10]
    knownCount: 0,
    unknownCount: 0,
    appearCount: 0,
    lastSeenTimeStamp: 0,
    nextReviewTime: 0
}
```

### 9.2 采样算法（`getRandomWord()` 三层权重）

**第一层 — 词库过滤：**
```javascript
// 按 state.wordPool 过滤候选词库
if (pool === FULL)     → wordDB 全部
if (pool === SUPER_HF) → wordDB 中属于 SUPER_HF_WORDS 集合的
if (pool === PHRASES)  → phraseDB (映射为临时 word 对象，id >= 10000)
if (pool === KEYWORDS) → keywordsDB 中所有群组涉及的 wordIds 对应 wordDB 子集
```

**第二层 — 可用词过滤：**
```javascript
// 排除 recentWordsWindow 中的词（20词环形防重队列）
// 排除 nextReviewTime 未到的词（尚未到复习时间的跳过）
pool.filter(word => !state.recentWordsWindow.includes(word.id) && Date.now() >= word.nextReviewTime)
```

**第三层 — 加权随机采样：**
```javascript
for each word:
    BaseWeight = Math.max(1, 11 - masteryScore)   // 熟练度越低权重越高
    TimeFactor = 1 + (now - lastSeenTimeStamp) / (24h)
    weight = BaseWeight × TimeFactor

汇总 totalWeight → random × totalWeight → 按权重落点选中
```

选中后：将该词 id 推入 `recentWordsWindow`（超过 20 则 shift 最早记录）。

### 9.3 熟练度更新矩阵（`updateMastery()`）

| 操作 | masteryScore 变化 | nextReviewTime 计算 | 附加动作 |
|------|-------------------|-------------------|---------|
| K（认识） | `+1`（上限 10） | `now + masteryScore × 30min` | `state.knowCount++` |
| J（不熟） | `-2`（下限 -10） | `now + 90s`（紧急回流） | `state.unfamiliarCount++` |

每次操作后更新 `appearCount++`、`lastSeenTimeStamp = now`，并调用 `saveProgress()`（3 秒节流写入 localStorage）。

### 9.4 短语/关键词的 masteryScore 处理

短语 (`id >= 10000`) 和关键词引用词 **不参与** 熟练度系统。`updateMastery()` 中对 `id >= 10000` 的直接 return。训练时也不会向 localStorage 写入这些临时对象的进度。

## 十、数据持久化策略

### 10.1 localStorage 存储键

| 键名 | 内容 | 写入频率 | 位置 |
|------|------|---------|------|
| `adhd-word-progress` | 每个单词的 masteryScore / appearCount / 时间戳 | **3 秒节流**（防 K/J 连击 I/O 阻塞） | `storage.js → saveProgress()` |
| `adhd-unfamiliar-words` | 不熟词 ID 数组 `[1, 5, 23, ...]` | **即时写入**（变化即落盘） | `storage.js → saveUnfamiliarWords()` |

### 10.2 进度存储格式

```json
// adhd-word-progress — 只存控制指标，不存完整单词
[
    { "id": 1, "masteryScore": 3, "appearCount": 12, "lastSeenTimeStamp": 17..., "nextReviewTime": 17... },
    { "id": 2, "masteryScore": -5, "appearCount": 8, "lastSeenTimeStamp": 17..., "nextReviewTime": 17... }
]
```

### 10.3 不熟词库生命周期

```
启动 → loadUnfamiliarWords() 从 localStorage 加载 → 写入 state.unfamiliarWords + 同步 unfamiliarCount
训练中按 J → addToUnfamiliar() → state 即时 push + saveUnfamiliarWords() 即时落盘
词库中按 J → markSelectedAsUnfamiliar() → 同上
不熟库中按 J → removeFromUnfamiliar() → state filter + saveUnfamiliarWords()
导出 → downloadUnfamiliarWords() → Blob → <a> 下载 .json
导入 → importUnfamiliarWords() → FileReader → 解析 → 合并去重 → saveUnfamiliarWords()
```

### 10.4 节流机制（saveProgress）

```javascript
let saveTimeout = null;
export function saveProgress() {
    if (saveTimeout) clearTimeout(saveTimeout);
    saveTimeout = setTimeout(() => {
        localStorage.setItem(STORAGE_KEY_PROGRESS, JSON.stringify(progressData));
        saveTimeout = null;
    }, 3000);
}
```

每次 `updateMastery()` 都调用 `saveProgress()`，但实际写入仅在最后一次调用的 3 秒后执行一次，保证高频 K/J 连击不会导致 I/O 卡帧。

## 十一、技术防御层

### 11.1 playSessionID 异步会话锁

**目的：** 防止 TTS 语音合成回调在用户切词/暂停后继续执行，导致脏数据注入 UI。

```javascript
// state.js
export function invalidateSession() {
    state.playSessionID++;           // 单调递增
    clearAllTimers();                // 清定时器桶
    try { speechSynthesis.cancel(); } catch(e) {}
}

// audio.js — speakWordAsync
const capturedSessionID = state.playSessionID;
utterance.onend = () => {
    if (state.playSessionID !== capturedSessionID) {
        resolve({ status: "aborted" });  // 过期会话 → 熔断
        return;
    }
    resolve({ status: "success" });
};
utterance.onerror = () => {
    resolve({ status: "error" });  // 错误不抛异常，防止卡死状态机
};
```

### 11.2 定时器 GC 托管桶

**规则：** 禁止孤立的 `setTimeout`。所有定时器必须通过 `safeScheduleTimer()` 创建并注册到 `state.activeTimersPool`。任何状态切换必须先调用 `clearAllTimers()`。

```javascript
export function safeScheduleTimer(callback, delay) {
    const timerId = setTimeout(() => {
        state.activeTimersPool = state.activeTimersPool.filter(id => id !== timerId);
        callback();
    }, delay);
    state.activeTimersPool.push(timerId);
}

export function clearAllTimers() {
    state.activeTimersPool.forEach(id => clearTimeout(id));
    state.activeTimersPool = [];
}
```

### 11.3 多键并弹输入互锁

**目的：** 防止用户 0.1 秒内狂按 K/J 导致同一词被重复刷分、状态机堆叠。

```javascript
// core.js — transition(event)
if (state.isProcessingInput && event !== 'SPACE') return;  // Space 在白名单内，总能响应暂停
state.isProcessingInput = true;

// 状态转移逻辑...

// 每个状态 handler 末尾：
state.isProcessingInput = false;
```

### 11.4 浏览器 TTS 物理极限补偿

| 档位 | rate 实际值 | 策略 |
|------|-----------|------|
| 1x | 0.8 | 正常 |
| 2x | 1.5 | 快 |
| 3x | **2.2**（硬封顶） | Chrome 超 2.5 静音 → 用 2.2 + 压缩 gap 模拟 3x |
| 5x | 不调用 TTS | 纯 `safeScheduleTimer` 文字闪现，绕过浏览器外设限制 |

### 11.5 防 GC 回收

浏览器可能在语音朗读期间 GC 回收 `SpeechSynthesisUtterance` 对象导致回调丢失。解决方案：

```javascript
window.globalAudioGarbageCollectorBucket = utterance;  // 挂载到全局，阻止 GC
```

### 11.6 防御性检查汇总

| 场景 | 防御措施 |
|------|---------|
| 切换模式 / 调速 | `invalidateSession()` → ID++ → 清定时器 → speechSynthesis.cancel() |
| 训练 → L/U 词库 | 同上 + 状态强制 IDLE |
| 空词库选中 | `if (pool.length === 0) pool = wordDB` |
| 所有可用词被防重排除 | `if (availableWords.length === 0) return pool[随机]` |
| TTS 不可用 | `if (!('speechSynthesis' in window)) resolve({status:"not_supported"})` |
| localStorage 损坏 | try/catch + console.warn，不进崩溃 |
